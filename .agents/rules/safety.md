# 安全护栏

定义 Agent 必须遵守的安全约束，违反即阻塞。

## 密钥与凭证

### 硬性禁止
- 禁止在任何代码文件中硬编码密钥、Token、密码
- 禁止在注释中记录凭证
- 禁止将 `.env` 文件提交到仓库
- 禁止在日志中输出敏感信息（Token、密码、身份证号、手机号）

### 正确做法
- 使用环境变量：`os.getenv("API_KEY")`
- 使用密钥管理服务（如 Azure Key Vault、AWS Secrets Manager）
- 日志输出使用脱敏：`"token: ***"` 形式

## 数据访问

### 权限最小化
- 数据库连接使用最小必要权限
- 禁止使用 sa/root 等高权限账户

### 敏感数据处理
- 用户数据必须加密存储
- API 响应中的敏感字段必须过滤
- 股票代码、手机号等半敏感信息脱敏输出

## 网络安全

### 请求安全
- 所有外部 API 调用使用 HTTPS
- 设置合理的 timeout 防止资源耗尽
- 实现重试机制时设置最大重试次数（≤ 3）

### 代理与隧道
- 禁止在代码中硬编码代理配置
- 代理设置通过环境变量控制

## 错误处理

### 信息泄露
- API 错误信息对外部隐藏内部实现细节
- 用户可见的错误使用友好文案，不暴露堆栈
- 内部日志记录完整堆栈信息

### 异常传播
- 禁止让未处理的异常直接暴露给用户
- 所有可能的异常必须有处理或记录

## 数据库

### SQL 注入
- 禁止使用字符串拼接构建 SQL
- 必须使用参数化查询或 ORM

### 事务
- 写操作必须考虑事务边界
- 长时间事务必须拆分

## 文件操作

### 路径安全
- 禁止直接拼接用户输入到文件路径
- 使用 `pathlib` 处理路径，禁止 `os.system` 执行 shell 命令

### 临时文件
- 临时文件使用 `tempfile` 模块
- 临时文件使用后必须清理

## 容器与部署

### 镜像安全
- 禁止在 Docker 镜像中存储密钥
- 使用多阶段构建减少镜像体积
- 基础镜像使用官方最小镜像

### 运行时
- 以非 root 用户运行容器
- 设置合理的资源限制（CPU、内存）

## 审查要求

以下改动必须由 Reviewer Agent 明确审查安全性：

- 认证/授权相关改动
- 数据库 schema 变更
- 外部 API 集成改动
- 配置文件变更
- 新增依赖项

## 禁止事项

- 禁止绕过安全检查
- 禁止在 TODO 中遗留密码占位符（如 `// TODO: set real key`）
- 禁止禁用 SSL/TLS 验证
- 禁止在生产环境使用 debug 模式
