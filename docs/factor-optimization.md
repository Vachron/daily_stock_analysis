# 因子优化方案：IC/PBO/DSR 四层验证体系

> 状态：已批准，待实施
> 最后更新：2026-04-26
> 关联代码：`src/core/strategy_signal_extractor.py`、`src/core/strategy_optimizer.py`、`src/core/market_regime.py`

## 背景

当前选股系统存在以下核心问题：

1. **因子有效性未量化**：21 种策略的信号评分基于人工设定阈值，缺乏统计意义上的有效性验证（IC 检验）。
2. **过拟合风险未检测**：策略权重优化仅基于历史胜率，未评估过拟合概率（PBO）和多重检验偏差（DSR）。
3. **因子冗余严重**：均线组、超卖组、形态组、突破组内部高度相关，冗余因子虚增 IC 信心。
4. **因子时效性未知**：不同因子的最优持有周期不同，但当前统一使用同一评分框架。
5. **缺乏动态监控**：因子失效只能靠人工发现，没有 IC 衰减预警和自动降权机制。

## 目标

- 建立从因子有效性到过拟合检测到动态监控的完整验证流水线
- 将 21 个策略去冗余为 8-10 个独立因子
- 实现因子健康度的每日自动体检
- 消除未来函数对验证结果的污染

## 第一层：因子有效性（IC）

### Rank IC 计算

**定义**：因子分数与未来收益率的 Spearman 秩相关系数。

```python
def compute_rank_ic(factor_scores: pd.Series, future_returns: pd.Series) -> float:
    return factor_scores.rank().corr(future_returns.rank(), method='spearman')
```

**关键约束**：
- 因子分数基于 T-1 日数据计算（避免当日收盘价的前视偏差）
- 未来收益率从 T 日开盘到 T+N 日收盘
- 最少需要 60 个交易日的滚动样本

**IC 判定标准**：

| IC 绝对值 | 判定 | 动作 |
|-----------|------|------|
| < 0.02 | 无效 | 废弃 |
| 0.02 ~ 0.03 | 微弱 | 观察期 |
| 0.03 ~ 0.05 | 有效 | 保留 |
| > 0.05 | 强效 | 高权重 |

### IC 衰减曲线

计算 1日/3日/5日/10日/20日 的 IC，绘制衰减图：

| 因子类型 | 典型衰减特征 | 最优持有周期 |
|----------|-------------|-------------|
| 动量/反转 | IC 在 3-5 日内最强，20 日后衰减为零 | 短周期（3-5日） |
| 价值/质量 | IC 在 20 日后才显现，5 日内可能为负 | 长周期（20日+） |
| 技术/形态 | IC 在 5-10 日内最强 | 中周期（5-10日） |

**规则**：因子使用时必须匹配其最优持有周期，短周期因子不应用于长周期持仓。

### 因子单调性检验（分层回测）

将股票按因子分数分为 5 组（quintile），分别计算每组未来 N 日平均收益：

- **单调递减** Q5 > Q4 > Q3 > Q2 > Q1 → 因子有效
- **极端值驱动** Q5 >> Q1 但中间层乱序 → 因子只捕捉了极端值，需改造
- **无单调性** 各组收益无规律 → 因子无效

### 因子正交性检验

计算因子间 IC 的相关系数矩阵：

| 冗余组 | 策略 | 共享核心逻辑 | 预期 IC 相关性 |
|--------|------|-------------|---------------|
| 均线组 | `bull_trend` + `ma_golden_cross` + `multi_golden_cross` | 均线排列/交叉 | > 0.7 |
| 超卖组 | `rsi_reversal` + `bollinger_reversion` + `shrink_pullback` | 跌多了买 | > 0.6 |
| 形态组 | `morning_star` + `w_bottom` + `one_yang_three_yin` | 底部反转 | > 0.5 |
| 突破组 | `volume_breakout` + `turtle_trading` + `limit_up_pullback` | 突破/新高 | > 0.6 |

**规则**：两个因子 IC 相关系数 > 0.6 时，只保留 IC 更高的那个。预计去冗余后保留 8-10 个独立因子。

### 因子换手率

每次选股运行时，记录因子排名的变化：

```
换手率 = |T日排名与T-1日排名不同的股票数| / 总股票数
```

**规则**：
- 换手率 > 50% 的因子需要 IC > 0.05 才值得使用（交易成本补偿）
- 换手率 < 20% 的因子 IC > 0.03 即可使用

## 第二层：过拟合检测（PBO + DSR）

### PBO（Probability of Backtest Overfitting）

基于 CSCV（Combinatorially Symmetric Cross-Validation）方法：

1. 将历史数据分为 S 个等长子区间（默认 S=16）
2. 枚举所有 C(S, S/2) 种训练/测试组合
3. 对每种组合，计算训练集最优策略在测试集的表现排名
4. PBO = 训练集最优但测试集排名低于中位的比例

**判定**：
- PBO < 0.3：低过拟合风险
- 0.3 ≤ PBO < 0.5：中等风险，需关注
- PBO ≥ 0.5：高过拟合风险，废弃策略

### DSR（Deflated Sharpe Ratio）

对夏普比率进行多重检验校正：

```
DSR = Φ( (Ŝ - μ_Ŝ) / σ_Ŝ )

其中：
Ŝ = 观测到的夏普比率
μ_Ŝ = 最高夏普比率的期望（考虑试错次数 N）
σ_Ŝ = 夏普比率的标准差
N = 已测试的策略总数
```

**判定**：
- DSR > 0.95：统计显著
- 0.5 < DSR ≤ 0.95：弱显著，需更多数据
- DSR ≤ 0.5：不显著，可能是数据挖掘结果

### Walk-forward 验证

使用滚动窗口而非固定 split：

```
窗口1: [1..60] 训练 → [61..70] 测试
窗口2: [11..70] 训练 → [71..80] 测试
窗口3: [21..80] 训练 → [81..90] 测试
...
```

**优势**：避免固定 split 的边界效应，更贴近实际交易场景。

### 参数敏感性分析

对策略阈值微调 ±10%，观察结果是否稳定：

- 如果阈值从 0.7 变为 0.63 或 0.77，IC/胜率剧烈波动 → 过拟合
- 如果阈值微调后结果稳定 → 策略鲁棒

## 第三层：动态监控与轮换

### 每日因子体检

每日选股运行后，自动执行：

1. **IC 扫描**：计算每个因子最近 20 日的滚动 IC
2. **PBO 重算**：使用最近 120 日数据重算 PBO
3. **IC 衰减速度**：比较 5 日 IC 均值与 20 日 IC 均值

### IC 衰减速度监控

| 衰减模式 | 5日IC均值 vs 20日IC均值 | 动作 |
|----------|------------------------|------|
| 稳定 | 差异 < 20% | 正常 |
| 缓慢衰减 | 差异 20%-50% | 降低权重 30% |
| 快速衰减 | 差异 > 50% | 紧急降权 70%，触发人工审查 |
| 反转 | 5日IC符号翻转 | 立即暂停，标记异常 |

### 因子拥挤度监控

通过因子排名的集中度衡量：

```
拥挤度 = Top N 股票在因子排名中的 Herfindahl 指数
```

- 拥挤度突增 → 大量资金涌入同一因子 → 收益将被套利殆尽 → 预警

### 集体衰减检测

- > 50% 因子 IC 同时下滑 → 市场结构性变化 → 风险警报
- 全部因子 IC < 0.02 → 市场可能进入无效状态 → 建议降低仓位

### 市场状态适配（Regime-conditional IC）

不同市场状态下因子的 IC 不同：

| 市场状态 | 有效因子类型 | 无效因子类型 |
|----------|-------------|-------------|
| 牛市 | 趋势/突破 | 反转/均值回归 |
| 熊市 | 反转/价值 | 趋势/动量 |
| 震荡 | 均值回归/箱体 | 突破/趋势 |

当前 `market_regime.py` 已有状态识别，需增加 IC 的 regime-conditional 计算。

## 第四层：因子改造与补充

### 量价融合改造

对 IC 有效但 PBO 偏高的因子，增加成交量确认：

| 原因子 | 融合方式 | 预期效果 |
|--------|---------|---------|
| `bull_trend` | 均线多头 + 成交量 > MA(vol,20) | 过滤假突破 |
| `ma_golden_cross` | 金叉 + 交叉日放量 | 提高信号可靠性 |
| `bollinger_reversion` | 触下轨 + 缩量（非恐慌性下跌） | 区分恐慌和超卖 |
| `volume_breakout` | 突破 + 量比 > 2.0 | 强化确认 |

### Alpha101 纯数学因子补充

增加不依赖主观判断的纯数学因子：

| 因子 | 公式 | 逻辑 |
|------|------|------|
| 日内振幅比 | `(C-O)/(H-L)` | 上影线/下影线强度 |
| 量价背离 | `sign(delta(volume,5)) * (-delta(close,5))` | 放量下跌 → 看空 |
| 波动率调整动量 | `close/delay(close,5) / (std(close,20)+1e-8)` | 动量除以波动率 |
| 成交量加权反转 | `-1 * correlation(rank(delta(close,5)), rank(volume), 5)` | 量价背离反转 |

**优势**：纯数学表达式无主观参数，天然抗过拟合。

### 因子最优持有周期匹配

基于 IC 衰减曲线，为每个因子标注最优持有周期：

- 短周期因子（3-5日）：用于日内/短线策略
- 中周期因子（5-10日）：用于波段策略
- 长周期因子（20日+）：用于趋势/价值策略

选股时，根据用户设定的持有周期偏好，自动筛选匹配的因子。

### 交易成本约束

```
最小可接受 IC = max(0.03, 换手率 * 单边交易成本 * 2 / 平均持仓收益)
```

高换手率因子需要更高的 IC 才能在扣除交易成本后盈利。

## 风险与约束

| 风险 | 说明 | 缓解措施 |
|------|------|----------|
| IC 计算依赖历史数据 | 数据不足时 IC 不可靠 | 最小样本量门控（≥200 交易日） |
| PBO 对样本量敏感 | 小样本下 PBO 不可靠 | 样本不足时标注"待验证"而非直接废弃 |
| 因子拥挤度误报 | 偶发性集中度上升 | 连续 3 日超阈值才触发预警 |
| 未来函数污染 | 前复权数据导致 IC 虚高 | 数据质量门控层强制检查 |
| 幸存者偏差 | 退市股票数据缺失 | 标注回测期间退市股票，计算偏差影响 |
| DSR 的 N 值设定 | 试错次数难以精确统计 | 保守估计 N = 已测试策略数 × 3 |

## 验证方式

- [ ] IC 计算引擎通过单元测试（使用已知数据验证 Spearman 相关系数）
- [ ] PBO 计算通过对照测试（使用已知过拟合策略验证 PBO > 0.5）
- [ ] DSR 计算通过对照测试（使用随机策略验证 DSR < 0.5）
- [ ] 因子正交性矩阵输出正确（冗余组 IC 相关性 > 0.6）
- [ ] 因子单调性检验输出正确（Q5 > Q4 > Q3 > Q2 > Q1）
- [ ] IC 衰减曲线与预期一致（动量因子短周期 IC 高，价值因子长周期 IC 高）
- [ ] 动态监控在 IC 衰减时正确触发降权
- [ ] 量价融合改造后 PBO 下降
- [ ] Alpha101 因子 IC > 0.03 且 PBO < 0.3
